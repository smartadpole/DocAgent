# 决策条目模板

> 用于 `[[projects/decisions]]` 的单条正式决策记录。默认放在单页结构的“正式决策记录”或“已覆盖 / 历史决策”区里；如果历史记录缺少某些信息，不补造不存在的选项、优劣或风险。

### 在这里直接写一句话决策标题

- **日期**：YYYY-MM-DD
- **状态**：默认写 `生效`；如果是历史回填可改成 `已覆盖` 或 `已纠正`
- **背景**：先交代这次为什么要拍板、当前冲突或约束是什么。
- **要决策什么**：
  1. 先写本轮真正需要拍板的核心问题。
  2. 如果不止一个待拍板点，继续补，但保持同一组决策主题。
- **可选项**：
  1. **方案 A**：一句话说明方案内容。
  2. **方案 B**：一句话说明方案内容。
- **最终决策**：
  1. 先写最终选择了哪一项。
  2. 再写当前正式生效的口径、边界和启用条件。
- **影响**：
  1. 先写会直接受影响的主页面、主流程或主范围。
  2. 如果有后续前提、显式重开条件或同步动作，也写在这里。
- **各自优劣**：
  1. **方案 A 优点**：写为什么它值得考虑。
  2. **方案 A 缺点**：写为什么它没有被选。
  3. **方案 B 优点**：写为什么它值得考虑。
  4. **方案 B 缺点**：写为什么它没有被选，或写它的代价。
- **风险点**：
  1. **选择方案 A 的主要风险**：写如果选它，最大的风险是什么。
  2. **选择方案 B 的主要风险**：写如果选它，最大的风险是什么。

## 使用说明

- 详细流程见 [[WORKFLOW]]，页面职责见 [[projects/STRUCTURE]]。
- [[projects/decisions]] 默认是单页入口：顶部用数字编号条目做当前生效摘要，正文里的正式条目供页内跳转回看。
- 摘要区默认写成“主题：决策。影响：摘要”；主题可直接挂页内跳转，决策和影响都只写一句摘要，不在摘要区罗列影响文件。
- 正式决策标题默认直接用稳定的一句话标题，不额外加编号；摘要和交叉引用直接链接标题即可。
- 默认顺序固定为：`**背景**`、`**要决策什么**`、`**可选项**`、`**最终决策**`、`**影响**`、`**各自优劣**`、`**风险点**`。
- `**最终决策**` 和 `**影响**` 必须放在比较和风险之前，先让读者知道“最后怎么定、会影响什么”。
- 标题尽量稳定，把日期写进 `**日期**` 字段，不要把日期写进标题。
- 只要使用“标签：内容”的展开式写法，冒号前标签统一加粗，不要写成裸标签。
- 决策页记录正式拍板和比较过程，不重复抄整份需求或整份设计。
