# Memory 开源工具

这页记录“分层 memory 系统”相关的开源工具研究。

主入口：[[projects/design/memory/README]]

## 结论先行

现成开源工具是有的，而且已经不少。

但更准确地说：

- “记忆存储”已经比较成熟
- “记忆治理”仍然需要自己补

也就是说，能找到开源底座，但很难直接找到一套原样覆盖“分层、晋升、冲突治理、多人协作统一口径”的完整系统。

## 候选工具

### Mem0

定位：长期记忆底座

更适合承接：

- 对话压缩
- 记忆抽取
- 记忆检索
- 降低 token 和延迟

适合当：

- memory service
- 通用记忆引擎

### Zep

定位：上下文工程 / 关系型共享脑

更适合承接：

- 项目级上下文
- 实体关系
- 多源上下文汇总
- 时间演化

适合当：

- 项目共享脑
- 关系上下文层

### Letta

定位：有状态 agent 运行时

更适合承接：

- 长期有状态 agent
- 跨会话状态保持
- 持续协作型 agent

适合当：

- agent 容器
- memory-aware runtime

## 怎么选

### 如果目标是尽快做出可用版

优先考虑：

- Mem0 或 Zep 作为底层记忆组件

然后自己补：

- 分层 schema
- 晋升规则
- conflict 状态机
- decision / policy 层

### 如果目标是做完整 agent 操作系统

优先考虑：

- Letta + 自己的 policy / memory orchestration

## 对当前项目的意义

这份调研的关键结论是：

- 开源项目可以提供“存”和“取”的基础能力
- 真正的差异化仍然在治理层

后续如果真的落地，不要期待某个现成项目直接替代：

- 分层规则
- 冲突升级机制
- 多人协作统一口径
- policy 优先级控制

## 当前判断

如果是 V1，最现实的做法是：

- 先用现成工具承接 memory engine
- 把真正关键的治理逻辑留在当前系统自己的文档、决策和规则层里

一句话总结：

现成开源有，但你真正要做的核心不是“记忆仓库”，而是“记忆编排系统”。
